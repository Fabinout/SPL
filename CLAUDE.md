## Project Overview

This repository contains the specification for **SPL (Structured Parenthesized Language)** — a low-level, stack-based virtual machine language designed to be easily generated by LLMs and trivially parsed. The full spec is in `SPL-spec-fr.md` (v1.0, written in French).

## Project Structure

```
SPL-spec-fr.md              — Language specification (authoritative, French)
examples/                   — Example SPL programs (documentation only)
vm-python/                  — Python VM implementation
  spl_asm.py                — Two-pass assembler (.spl → .rom)
  spl_vm.py                 — Bytecode interpreter
  tests/
    test_all.spl            — Self-contained test (22 tests, every opcode)
    run_tests.py            — Automated test runner (8 checks)
```

## Commands

```bash
# Assemble .spl to .rom bytecode
python vm-python/spl_asm.py <input.spl> [output.rom]

# Run a .rom in the VM
python vm-python/spl_vm.py <program.rom>

# Run all tests
python vm-python/tests/run_tests.py
```

## Architecture

- **`vm-python/spl_asm.py`** — Two-pass assembler. Pass 1 collects labels and byte offsets. Pass 2 emits bytecode with resolved addresses. Validates argument types (rejects labels for `load`/`store`, range-checks values/ports/addresses).
- **`vm-python/spl_vm.py`** — Bytecode interpreter. Harvard architecture: code (ROM) and data (64 KiB RAM) are separate address spaces. Implements console I/O, RNG, 32-bit timer with latch, and SYSCTL_CAPS. Conditional jumps (`jump-if-zero`, `jump-if-not-zero`) pop the tested value.

## SPL Language Quick Reference

### Execution Model
- **Stack-based VM** with a work stack (min 256 entries), a return stack (min 64 entries), and 64 KiB linear memory
- All values are **8-bit unsigned** (mod 256); bytecode addresses are **big-endian**
- Halt on: `(halt)`, PC past end of bytecode, stack underflow/overflow, OOB memory access
- Division by zero returns 0

### Instruction Set
- **Halt**: `halt` (0x00)
- **Stack**: `push` (0x01), `drop`, `dup`, `swap`, `over`
- **Arithmetic/Logic**: `add`, `sub`, `mul`, `div`, `mod`, `and`, `or`, `xor`, `not` (bitwise)
- **Memory**: `load` (0x20), `store` (0x21) — direct addressing only, numeric 16-bit constants
- **Control Flow**: `jump`, `jump-if-zero`, `jump-if-not-zero`, `call`, `return` — accept label names or literal addresses
- **I/O**: `in` (0x40), `out` (0x41) — 8-bit port number
- **Pseudo**: `label` — emits no bytes, resolves to byte offset
- Binary operand order: second-from-top is left, top is right

### Port-Mapped I/O (currently implemented in vm-python)
- `0x01`–`0x03`: Console (data W, status R, flush W)
- `0x10`: RNG (R)
- `0x11`–`0x14`: Timer 32-bit ms (R, latch on B3 read)
- `0xFF`: SYSCTL_CAPS (R)

### Implementation Notes
- Labels resolve to 16-bit byte offsets; forward references allowed; names must be unique
- `push` values are strictly 0–255
- Console buffers output, flushes on `\n` or `(out 0x03)`
- VM currently supports Core I/O profile only. Keyboard, video, audio, and mouse ports return 0 / are silently ignored
